О ПроЕкторе

ПроЕктор - это система, позволяющая организовывать совместную работу нескольких людей над одними и темu же проектами. Система позволяет разбивать пользователей на группы, создавать новые проекты, выдавать задания пользователям в рамках того или иного проекта, т.е. обеспечивает практически полный пакет инструментов управления проектами.

Подключение через moder:qwerty and user:qwerty

{НедоТЗ ПроектЕктора}

Группы сотрудников
- по направлению руководства
  • администратор
  • руководитель проекта
  • модератор
  • пользователь
- по характеру деятельности
  • художник
  • сценарист
  • программист
  • тестер
  • моделлер
  • композитор

Группы по направлению руководства
Администратор – имеет полные права
Руководитель проекта – создаёт задания и назначает их пользователям внутри своего проекта, редактирует проект
Модератор – создаёт и редактирует любые проекты, задания в них (???)
Пользователь – оставляет комментарии, просматривает проекты и задания (согласно уровня доступа), прикрепляет файлы, отправляет сообщения

Группы по характеру деятельности
Пользователи, в зависимости от группы, в которой состоят, могут получать соответствующие задания. Пользователи могут находиться сразу в нескольких группах.

Имеем страницы:
- главная (она же Мои задачи)
- сотрудники
- проекты
- сообщения

Профиль (profile)
Тут сотрудник может прочитать личные сообщения и проверить свои задания, отредактировать личные данные. На чужой – показывает информацию о сотруднике, его задания. Можно оставить ему сообщение.

Сотрудники (user)
Список всех сотрудников компании. При выборе переходим на страницу профиля выбранного сотрудника.

Проекты (project)
По умолчанию показывает список проектов (согласно доступа). Можно выбрать проект для просмотра (отображается полная информация, задания, файлы(?))

Задания (task)
Показывает полную информацию о конкретном задании (в т.ч. список файлов). Здесь же ведётся обсуждение задания

Сообщения (mess)
Открывает личные сообщения сотрудника или полностью выбранное сообщение (+ форма ответа)

Значения для db.uzrz.state:
-2 - требуется активация по мылу и по-админски
-1 - требуется активация по мылу
0 - требуется активация по-админски
1 - пользователь
2 - модератор
3 - админ

Значения для db.mssgz.mget:
0 - непрочитано
1 - прочитано
2 - удалено отправителем
3 - удалено получателем

Значения для db.pstz.postintask:
0 - проект
1 - задание
2 - группа
3 - новость
4 - событие
5 - файл
6 - баг
